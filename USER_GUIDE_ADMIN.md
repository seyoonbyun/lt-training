# BNI Korea Leadership Training 2025 - 어드민 사용자 가이드

## 🎯 시스템 개요

### 관리자 권한 및 역할

**BNI Korea Leadership Training 2025** 시스템의 관리자는 구글 시트를 통해 모든 교육 프로그램을 관리하며, 웹 애플리케이션과 실시간 양방향 동기화를 통해 효율적인 운영이 가능합니다.

#### 🚀 관리 시스템의 핵심 특장점

1. **구글 시트 중심 관리**
   - 별도 관리자 페이지 불필요
   - 익숙한 스프레드시트 환경에서 모든 관리

   - A-I 열 구조로 체계적 데이터 관리
   - 결제 상태 기반 신청자 수 자동 계산

3. **유연한 콘텐츠 관리**
   - 강의실 링크 실시간 업데이트
   - 과목 상태 즉시 변경 (접수중/마감)
   - 교육 일정 및 장소 정보 관리

## 🔧 핵심 관리 작업

### 1. 신청자 관리

#### 신청 데이터 확인
- **자동 기록**: 웹에서 신청 시 A-H열까지 자동으로 기록됨
- **실시간 반영**: 신청 즉시 시트에 새 행 추가
- **데이터 무결성**: 과목명(A열)은 자동으로 정확한 형식으로 입력됨

#### 결제 상태 관리 (I열) ⭐ 가장 중요
```
✅ "완료" 입력 시: 
   - 신청자 수에 포함됨
   - 웹사이트에 "신청자: XX명(결제완료기준)"으로 표시

⏳ 공백 또는 다른 값:
   - 신청자 수에 포함되지 않음
   - 결제 대기 상태로 처리
```

**주의사항:**
- I열에 "완료"라고 정확히 입력해야 함
- 대소문자, 띄어쓰기 주의
- 결제 확인 후 즉시 "완료" 상태로 변경

### 2. 프로그램 정보 관리

#### 강의실 링크 관리 (H열)
```
온라인 과목: https://timelog-bnikorea.com/
오프라인 과목: 해당 없음 (비워둠)
```

**변경 방법:**
1. 두 번째 시트의 H열에서 해당 과목 찾기
2. 새로운 링크 URL 입력
3. 저장 시 웹사이트에 즉시 반영

#### 장소 정보 관리 (I열)
```
예시:
- "온라인 강의실(익일 오후1시)"
- "스페이스 쉐어 삼성점 - 서울특별시 강남구 영동대로 513"
- "http://naver.me/GUGJFEBT" (지도 링크)
```

**변경 방법:**
1. 두 번째 시트의 I열에서 해당 과목 찾기
2. 장소 정보 수정
3. 링크인 경우 전체 URL 입력

### 3. 과목 상태 관리

#### 접수 상태 제어
```
접수중: true
마감: false
```

**변경 방법:**
1. 세 번째 시트에서 해당 과목명 찾기
2. 해당 과목 행에서 상태 값 변경
3. true: 신청 버튼 활성화
4. false: 신청 버튼 비활성화

**즉시 반영:**
- 상태 변경 시 웹사이트에서 5초 내 반영
- 마감 시 신청 버튼이 회색으로 비활성화됨

## 🚨 문제 해결 가이드

### 일반적인 문제들

#### 1. 신청자 수가 맞지 않을 때
**원인:** I열 결제 상태 표기 오류
**해결:**
1. 첫 번째 시트에서 I열 확인
2. "완료"가 아닌 다른 표기 찾기
3. 정확히 "완료"로 수정

#### 2. 강의실 링크가 작동하지 않을 때
**원인:** H열 URL 오류
**해결:**
1. 두 번째 시트에서 H열 확인
2. 전체 URL 형식으로 다시 입력
3. 예: `https://timelog-bnikorea.com/`

#### 3. 과목이 마감되지 않을 때
**원인:** 세 번째 시트 상태 설정 오류
**해결:**
1. 세 번째 시트에서 해당 과목 찾기
2. 상태를 `false`로 변경
3. 5초 후 웹사이트에서 확인

#### 4. 신청 데이터가 누락될 때
**원인:** 구글 시트 권한 또는 API 오류
**해결:**
1. 구글 시트 공유 설정 확인
2. 시트 편집 권한 확인
3. 필요 시 시스템 관리자에게 문의

### 비상 상황 대응

#### 시스템 점검 시
1. 세 번째 시트에서 모든 과목을 `false`로 설정
2. 웹사이트에 점검 안내 표시됨
3. 점검 완료 후 `true`로 복원

#### 대량 신청 처리 시
1. 일괄 신청 데이터 검토
2. I열 결제 상태 일괄 업데이트
3. 신청자 수 변화 모니터링
